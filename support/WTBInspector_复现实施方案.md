# WTBInspector 复现实施方案

> 基于论文：*WTBInspector: An Automated UAV-based Wind Turbine Blade Inspection System*（IEEE TFR, 2026）
> 作者：Yichuan Shi, Hao Liu, Haowen Zheng 等（中山大学 & 中国电建中南院）

---

## 一、项目概述

### 1.1 目标

复现 WTBInspector 的核心功能：无人机全自主风机叶片巡检系统，包括**风机参数自动估计（AWPE）**、**云台导向叶片表面跟踪（GOST）**、**实时图像质量优化（RIQO）**三个模块。

### 1.2 现有条件

- 硬件：DJI M300 RTK、Livox MID-360、DJI Zenmuse H20T、Intel NUC
- 资源：合作风电场可配合停机测试
- 团队：计算机学院本科/硕士生，具备 C++/Python 编程能力，无 ROS/无人机领域经验

### 1.3 总体评估

| 维度 | 评估 |
|------|------|
| 技术可行性 | 可行，核心算法均基于成熟库（PCL/ROS），论文有完整伪代码 |
| 硬件可行性 | 全部商用现货 |
| 人力需求 | 4-6 人 |
| 时间周期 | 13-16 个月 |
| 最大风险 | 实飞安全 + 算法工程化 + 无经验人员踩坑 |

## 三、技术栈说明

### 3.1 软件框架

| 组件 | 选型 | 说明 |
|------|------|------|
| 操作系统 | Ubuntu 22.04 | ROS 2 Humble 推荐版本 |
| 中间件 | ROS 2 Humble | PSDK 官方支持 ROS 2，PCL/Gazebo 均已适配 |
| 点云库 | PCL 1.12+ | RANSAC/KDTree/VoxelGrid/DBSCAN 均直接调用 |
| 仿真 | Gazebo + PX4 | 论文同款方案 |
| 无人机 SDK | DJI PSDK 3.x | 替代论文时期的 OSDK |
| LiDAR 驱动 | Livox ROS2 Driver | 社区版 `livox_ros2_driver` |
| 开发语言 | 核心算法 C++，工具脚本 Python | 性能敏感的路径用 C++ |

### 3.2 PSDK 说明（重要变更）

论文发表时使用 DJI OSDK，目前 M300 RTK 已统一到 **PSDK（Payload SDK）**。关键差异：

| | OSDK（论文） | PSDK（当前） |
|------|------|------|
| 连接方式 | USB/UART | SkyPort V2 / USB bulk |
| ROS 支持 | `dji_osdk_ros`（ROS 1） | 官方 `psdk_ros2`（ROS 2） |
| 速度指令 | `flight_control_setpoint_generic` | `JoystickCommand` / `VelocityAndYawRateCommand` |
| 相机控制 | 通过 Gimbal Manager 间接操作 | `Camera Manager` 直接操作曝光/变焦 |

**坐标系差异需注意：** PSDK 的速度指令使用 ENU 坐标系，PX4 仿真使用 NED 坐标系。需要在飞控接口层做统一封装，保证仿真和实飞用同一套 API。

### 3.3 Livox 兼容性注意

Livox MID-360 官方只提供 ROS 1 驱动（`livox_ros_driver`），ROS 2 版本为社区维护（`livox_ros2_driver`）。备选方案：Livox ROS 1 驱动 + `ros1_bridge` 桥接到 ROS 2。

### Phase 3：GOST 模块——叶片表面跟踪（第 6-8 月）

| 37 | 建立叶片局部坐标系 | u^λ_s（spanwise）= 叶片向量；u^λ_c（chordwise）= u_H 在 u^λ_s 正交面上的投影；u^λ_n（normal）= u^λ_s × u^λ_c | 论文 IV-B1, Fig.9 |

### 8.1 3D 打印保护壳

这是**最容易拖延硬件进度的环节**，不是因为设计难，而是因为迭代周期长。

**设计时必须考虑的约束：**

| 约束 | 说明 |
|------|------|
| NUC 散热 | M300 飞行时螺旋桨下洗气流有一定帮助，但保护壳不能完全密封。NUC 满载时功耗约 40W，必须在进风口和出风口留开口。建议在进气口加防尘网（风电场沙尘大） |
| NUC 开机方式 | NUC 装进壳子后外面按不到电源键。提前在 BIOS 里设 **"After Power Failure: Power On"**，这样 SkyPort 一供电 NUC 就自动开机。否则每次都要拆壳按电源 |
| 重量对称 | M300 挂载点在下沉式导轨上，壳子 + NUC + LiDAR 加起来约 2-3kg，要保证重心在无人机几何中心下方。设计前先用 SolidWorks 做质心分析 |
| LiDAR 视场遮挡 | MID-360 是 360° 水平 × 60°(-30°~+30°) 垂直 FOV。壳体和机臂不能挡到 LiDAR 的视场，尤其注意 M300 前臂在 LiDAR 前方 ±45° 方向有遮挡风险——仿真里就要验证 |
| H20T 云台活动空间 | H20T 云台需要 pitch ±120°、yaw ±320° 的旋转空间，壳子不能卡到云台 |
| 快拆设计 | 外场搬运时经常需要拆装，壳子与 M300 导轨的连接要方便拆装。不要用螺丝拧死 |
| 防雨能力 | 论文提到轻雨环境可工作。壳子接缝处加密封胶条，底部留排水孔（防止冷凝积水） |

**打印与迭代计划：**

| 步骤 | 时间 | 备注 |
|------|:--:|------|
| 第一版：验证尺寸 | 第 1-2 周 | 用便宜材料（PLA）打一个简版，不上设备，只验证安装孔位、导轨接口、整体尺寸是否对 |
| 第二版：装真设备 | 第 3 周 | 所有设备装进去，地面通电跑 30 分钟，红外测温看 NUC 是否过热（> 85°C 要加通风） |
| 第三版：优化+加固 | 第 4-5 周 | 用工程材料（ABS/尼龙/碳纤增强），加固关键受力点，加防尘网 |
| 最终版 | 第 6 周 | 预留 1-2 个备件 |

> **耗时估算：** 从设计到拿到可用壳子，预留 **6 周**。3D 打印一版需要 1-3 天，快递还要 2-3 天，反复改图反复等。

### 8.2 电缆与布线

| 事项 | 细节 | 踩坑后果 |
|------|------|----------|
| NUC 供电 | M300 SkyPort v2 的 SkyPort Power 输出是 **19.2V / 4.5A（约 86W）**。NUC11TNKi5 需要 19V DC，功率够但要注意：**启动瞬间电流尖峰可能触发 SkyPort 过流保护**。建议在电源线上串一个 NTC 热敏电阻做软启动，或者在 NUC 端加一个小电容缓冲 | 天上 NUC 突然掉电重启，整个系统失联 |
| MID-360 供电 | MID-360 需要 9-18V，SkyPort 可以直接供电。但它**没有自己的开关**——一上电就开始转。NUC 如果还没启动完就收到大量点云数据，可能导致 ROS 节点启动异常。代码里要做好"点云数据到达但系统未就绪"的容错 | ROS 节点 crash，起飞前需要重启 |
| 线缆长度 | 先装机再量线。所有线留 5cm 余量，不能绷太紧（飞行振动会拉扯接头）。但也不能太长，否则可能被螺旋桨下洗气流吹得乱晃 | 松脱 → 空中断电；过长 → 线缆缠桨 |
| 接头固定 | 每个 USB/电源接头处用 **热缩管 + 扎带** 双重固定。不要只用胶带——夏天风电场机舱附近温度可达 40°C+，胶带会脱胶 | 空中接头松动，数据中断 |
| 线缆标识 | 每根线两端贴标签（A 端 → B 端），不同电压的线用不同颜色热缩管区分 | 外场排故时拔错线 |
| 备用线缆 | 外场带一套备用线（电源线、USB 线、网线各 2 根） | 一根线坏了当天直接收工 |

### 8.3 NUC 配置细节

| 事项 | 操作 |
|------|------|
| BIOS：通电自启 | Advanced → Power → After Power Failure → **Power On**。否则装进壳子后每次开机都要拆壳 |
| BIOS：关掉睡眠 | 关掉所有 sleep/hibernate 选项。Ubuntu 进休眠后 ROS 全崩 |
| Ubuntu：关自动更新 | `apt remove unattended-upgrades`。外场没网，更新服务会在后台狂耗 CPU |
| Ubuntu：关 GUI | NUC 装的是 Ubuntu Server 或用 `systemctl set-default multi-user.target` 关掉桌面。省内存给 ROS |
| IP 固定 | NUC 的以太网口设固定 IP（如 192.168.1.100）。若用 USB 转以太网连 M300，确认 M300 端的 IP 段 |
| SSH 免密 | 配好 SSH key，外场笔记本直接 `ssh nuc@192.168.1.100` 就能上去 |
| 时间同步 | NUC 没有电池供电时每次断电系统时间会重置。写一个脚本：ROS 收到第一个 RTK GPS 消息时用 GPS 时间戳校正系统时间 |
| 磁盘空间 | 检查 `/var/log` 和 rosbag 存储路径。一次 30 分钟飞行录全部 topic 大约 30-50GB。确保 NUC 空余 ≥ 200GB |

### 8.4 DJI 固件与 SDK 版本兼容

这是整个硬件集成阶段**最容易踩的坑**。DJI 的版本管理非常严格。

| 组件 | 注意事项 |
|------|----------|
| M300 固件 | 必须和 PSDK 版本匹配。PSDK 3.x 要求 M300 固件 ≥ v04.00.0200。**拿到 M300 后第一件事：在 DJI Pilot 2 App 里查固件版本并截图** |
| 遥控器固件 | RC Plus 也要更新到匹配版本，否则 SkyPort 不给 PSDK 授权 |
| PSDK 版本 | 用最新的 PSDK 3.x。不要用 PSDK 2.x（API 不兼容） |
| SkyPort 授权 | PSDK 应用需要先在 DJI Developer 网站注册、申请 app key + app ID，把它们写死在代码里。**这个过程需要审批，提前 1-2 周申请** |
| Livox 固件 | MID-360 出厂固件可能很老，先用 Livox Viewer 连上检查，更新到最新 |
| H20T 固件 | 通过 DJI Pilot 检查并更新 |

**版本核验时间线（拿到硬件第一周必做）：**

```
Day 1: 所有设备开箱，记录序列号和当前固件版本（拍照存档）
Day 2: 把所有固件更新到最新稳定版
Day 3: NUC 装好 PSDK 的 sample 程序，上 M300 SkyPort 跑通 Hello World
       （确认 PSDK 能初始化、能收到飞行数据、能发 JoystickCommand）
Day 4: 如果 sample 跑不通，检查固件版本 → 降级/升级 → 重试
       这个循环可能反复 2-3 天
```

> **经验：** PSDK 的 sample 程序跑通之前，不要写一行自己的代码。sample 跑通了 = 链路没问题 = 可以在 sample 基础上改。

### 8.5 RTK 基站架设

| 事项 | 细节 |
|------|------|
| 架设位置 | RTK 基站（D-RTK 2）需要架在三脚架上，周围无遮挡，天空可见面积越大越好。风机旁有山/树林的话基站要架在开阔处 |
| 架设时机 | **RTK 冷启动搜星 + 收敛到 FIX 状态需要 5-15 分钟**。到达风电场第一件事就是架基站，而不是最后才架 |
| 基站与 M300 通信 | D-RTK 2 通过 OcuSync 与 M300 通信，有效距离约 2km（无遮挡）。如果风电场有多个风机且相距较远，基站可能需要搬位置 |
| RTK 状态监控 | 代码里订阅 PSDK 的 RTK 状态位：`RTK_FIX` 才飞自主模式，`RTK_FLOAT` 或 `NONE` 只许手动飞 |
| 备用方案 | 如果 RTK 基站出问题（充电不足、摔坏），M300 单靠 GPS 定位误差约 1-3m，这对近风机飞行非常危险。**RTK 不是锦上添花，是安全前提**。建议带一块备用基站电池 |

### 8.6 外场网络与远程操作

风电场通常手机信号都不好，要带自己的局域网。

| 事项 | 方案 |
|------|------|
| 地面站 ↔ NUC | NUC 开机后自启一个 Wi-Fi 热点（`create_ap` 或 `hostapd`），地面笔记本连这个热点 SSH 上去。不要依赖任何蜂窝网络或外网 |
| NUC ↔ M300 | 走 SkyPort 的 USB bulk 通道，不走网口（这是 PSDK 默认方式） |
| Livox ↔ NUC | MID-360 通过以太网口连 NUC（100Mbps），NUC 要有两个网口（一个内置、一个 USB 转 RJ45） |
| 日志回传 | 飞行结束后 SSH 上去 `scp` 拉 rosbag 文件。一次飞行 30-50GB，用千兆网大约 5-8 分钟。如果飞行间隔紧，带一个外置 SSD 直接插 NUC USB 拷 |
| 地面显示器 | 带一台便携显示器（或平板），用 `rqt` 或自写 dashboard 实时显示：当前状态机阶段、距风机距离、电池电量、RTK 状态、点云帧率 |

### 8.8 风电场协调清单

这些是出发前就要和风电场确认好的事，否则到了现场可能白跑。

| 确认事项 | 为什么重要 |
|----------|------------|
| 风机型号 + 塔筒高度 + 叶片长度 | 提前输入仿真环境做针对性测试 |
| 风机停机后偏航角是否可以调整？叶片停止角是否可以预约定？ | 论文提到这两个参数无法提前获知，但实际风电场操作人员可能可以手动设到某个大致角度。Y 形位置（叶片呈倒 Y）最不利于检查（论文也说了），**请操作人员尽量避开 Y 形** |
| 停机后风机是否完全锁死（偏航刹车 + 变桨刹车）？ | 论文明确要求两者都锁死。如果只锁了一个，风机在风中可能会动，非常危险 |
| 停机可分配的时间窗口 | 一台风机停一次 30 分钟，一天能停几台？有没有专人在机舱里操作锁机？ |
| 场区内是否有禁飞区或高压线？ | M300 内置的禁飞区可能不完整。风机之间常有架空输电线，让风场人员在地图上标出来 |
| 天气窗口 | 提前查预报：风速 ≤ 8m/s、无雨无雾。风电场的风速通常比天气预报大 1-2 级 |
| 进场手续 | 大多数风电场需要提前签安全协议、做入场安全教育。提前 1 周联系 |

### 8.10 软件部署工作流

每次代码更新后到能飞之间的步骤。这个流程如果不标准化，会反复出低级错误。

```
1. git push（实验室开发机）
2. ssh nuc → git pull → colcon build → 确认编译通过
3. 跑仿真回归测试脚本（≥ 10 个随机风机工况，全绿）
4. 给 NUC 上的二进制打版本号（git commit hash 写进 rosbag 的 /metadata）
5. 关机 → 把所有线缆拔掉再重新插紧 → 再开机（模拟运输后重新连接）
6. ssh 上去跑一个 "preflight check" 脚本：
   - PSDK 初始化成功？
   - LiDAR 点云有数据且帧率正常？
   - RTK FIX 状态？
   - 相机画面正常？
   - 电池 > 90%？
   - 磁盘剩余 > 50GB？
7. 全部通过 → 可以起飞
```

**仿真回归测试脚本的内容：**

```
for each 工况 in [偏航=-45,-15,0,15,45 × 倾角=3,5,6 × 停止角=0,60,120,180,240,300]:
    launch 仿真
    运行全流程
    if AWPE 任何参数误差 > 3×论文 MAE：
        报错并记录
    if GOST 导致碰撞（距离风机 < 5m）：
        报错
    输出通过/失败
```

### 8.11 常见踩坑速查

| 现象 | 最可能原因 | 排查顺序 |
|------|------------|----------|
| PSDK 初始化失败 | SkyPort 没授权 / 固件版本不匹配 | 1. 检查 DJI Pilot 里 SkyPort 是否识别 2. 检查固件版本 3. 重新申请 PSDK app key |
| NUC 不启动 | BIOS 没设通电自启 / SkyPort 供电不足 | 1. 拆壳外接显示器看 BIOS 2. 万用表量 SkyPort 输出电压 |
| Livox 没有点云 | IP 没配通 / 固件太老 | 1. `ping 192.168.1.200`（Livox 默认 IP）2. Livox Viewer 直连确认硬件正常 |
| 点云帧率很低（< 5Hz） | NUC USB 带宽不够 | MID-360 不要跟 H20T 共用同一个 USB 控制器。M300 SkyPort 的 USB 走的是内部 Hub，确认没有其他带宽大户 |
| RTK 一直 FLOAT 不 FIX | 基站搜星不足 / 基站到 M300 太远有遮挡 | 1. 移动基站到更开阔处 2. 等待 10 分钟 |
| 飞机悬停时漂移 > 0.5m | RTK 掉到 FLOAT / 指南针受风机金属结构干扰 | 1. 检查 RTK 状态 2. 增大与风机的最小安全距离 |
| rosbag 文件损坏 | NUC 非正常关机 | 每次飞行结束后先 `rosnode kill` 然后 `sync` 再拔电 |
| NUC 过热降频 | 壳子散热不够 | 1. 红外测温检查 2. 加通风口 3. 加小风扇 |

### 8.12 风电场第一天行动清单

```
07:00  到达风电场，找操作人员做安全交底
07:30  架 RTK 基站，确认搜星
07:45  组装无人机（上电池、检查桨叶、SD 卡格式化）
08:00  给 NUC 上电，SSH 连上跑 preflight check
08:15  飞手用手动模式起飞悬停 1 分钟，确认飞机状态正常
08:30  操作人员锁第一台风机
08:35  手动飞近风机，在 30m 距离悬停，收 LiDAR 点云
       检查点云质量：塔筒能看到吗？叶片能看到几根？
08:45  手动绕风机飞一圈，全程录 rosbag
09:00  降落，换电池，回实验室区域拷 rosbag 到笔记本
       立刻跑一遍 AWPE 离线处理，看在真实点云上能否提取出塔筒/叶片
```
